# Especificações do EPROM `27C256`

O **`27C256`** é um chip EPROM (Erasable Programmable Read-Only Memory) padrão de 32 KB (256-kilobit). É o chip de memória principal usado para armazenar calibrações personalizadas de combustível e ignição em ECUs Honda OBD0 e OBD1 modificadas (chipped).

---

## 1. Classificações Críticas de Velocidade

Os processadores internos das ECUs Honda leem dados a alta velocidade. Ao adquirir EPROMs 27C256, presta muita atenção ao tempo de acesso (classificação de velocidade) impresso no chip.

* **Requisito:** A ECU requer EPROMs com classificação de **200 nanossegundos (ns) ou mais rápidos**. 
* **Velocidades Compatíveis:** Chips de 90ns, 100ns, 120ns e 150ns funcionarão perfeitamente.

* **Descodificação de Etiquetas:** Os sufixos dos chips indicam a velocidade:
 

* `-90` ou `-9` = 90ns
 * `-12` ou `-120` = 120ns
 

* `-15` ou `-150` = 150ns
 * `-20` ou `-200` = 200ns
 
> [!WARNING]
> Evita chips com classificação mais lenta do que 200ns (como 250ns ou 300ns), pois causarão erros de sincronização de ponto (timing), acendendo a luz de verificação do motor (CEL) de forma fixa e impedindo o arranque do motor.

---

## 2. Equivalentes de Chips e Encapsulamento

* **Encapsulamento DIP-28:** O `27C256` está alojado num encapsulamento padrão de 28 pinos Dual In-line Package (DIP-28) com um espaçamento de pinos de 0,6 polegadas.

* **Chips OKI de Fábrica:** As ECUs Honda originais contêm um chip mask ROM pré-programado com a etiqueta **OKI 38256**. Isto é simplesmente um equivalente impresso de fábrica e de escrita única (write-once) de um `27C256` padrão.

---

## 3. Layout de Pinagem DIP-28

Abaixo está o mapeamento padrão de pinagem para uma EPROM 27C256:

```text
 +------()------+
 VPP |1 28| VCC (+5V Power)
 A12 |2 27| A14
 A7 |3 26| A13
 A6 |4 25| A8
 A5 |5 24| A9
 A4 |6 23| A11
 A3 |7 27C256 22| /OE (Output Enable)
 A2 |8 21| A10
 A1 |9 20| /CE (Chip Enable)
 A0 |10 19| D7 (Data Bit 7)
 D0 |11 18| D6
 D1 |12 17| D5
 D2 |13 16| D4
 GND |14 15| D3 (Data Bit 3)
 +--------------+
```

### Explicações dos Pinos:

* **A0–A14:** Linhas de Endereço (linhas de entrada usadas pelo CPU para selecionar bytes de memória específicos).

* **D0–D7:** Saídas de Dados (barramento de dados de 8 bits que devolve valores de bytes para o CPU).

* **VPP:** Pino de Tensão de Programação (normalmente requer 12.5V durante as operações de escrita em gravadores padrão).

* **VCC:** Pino de Fonte de Alimentação (+5V de alimentação lógica durante o funcionamento normal).

* **/CE (Chip Enable):** Seleciona ativamente o chip de memória quando colocado a nível lógico baixo (0V).

* **/OE (Output Enable):** Coloca ativamente os dados no barramento de dados quando colocado a nível lógico baixo.
