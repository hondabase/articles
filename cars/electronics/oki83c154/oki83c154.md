# Oki83C154

[MCU](/cars/electronics/mcu) used in [OBD0 MPFI](/cars/electronics/obd0mpfi) [ECU](/cars/electronics/ecu)s that has 16k on-chip code memory. Usually copy protected, but there are [Mcu Readers](/cars/electronics/mcu-readers). It is mostly 8051 compatible, and features a third timer like many other replacements. The main difference from a pure 8051 revolves around how the [A5](/cars/electronics/a5) instruction is handled. Can be replaced by a [Intel8051](/cars/electronics/intel8051)-compatible [MCU](/cars/electronics/mcu) ***iff*** [A5](/cars/electronics/a5)-free code is used. There are MANY 8051 compatible [ECU](/cars/electronics/ecu)s, and many of them are also in the same 40PDIP package as the 83C154, making replacing the [MCU](/cars/electronics/mcu) very easy. The Dallas 80C320, ATMEL 89C55, T89C51RD2 have all been tested by me (Dave).
