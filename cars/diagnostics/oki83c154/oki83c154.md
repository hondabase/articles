# Oki83C154

[MCU](/cars/rom/mcu) used in [OBD0 MPFI](/cars/diagnostics/obd0mpfi) [ECU](/cars/ecu/ecu)s that has 16k on-chip code memory. Usually copy protected, but there are [Mcu Readers](/cars/wiring/mcu-readers). It is mostly 8051 compatible, and features a third timer like many other replacements. The main difference from a pure 8051 revolves around how the [A5](/cars/diagnostics/a5) instruction is handled. Can be replaced by a [Intel8051](/cars/rom/intel8051)-compatible [MCU](/cars/rom/mcu) ***iff*** [A5](/cars/diagnostics/a5)-free code is used. There are MANY 8051 compatible [ECU](/cars/ecu/ecu)s, and many of them are also in the same 40PDIP package as the `83C154`, making replacing the [MCU](/cars/rom/mcu) very easy. The Dallas 80C320, ATMEL 89C55, T89C51RD2 have all been tested by me (Dave).
